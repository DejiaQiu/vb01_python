# 功能需求单

## 目标
- 将钢丝绳松动专项诊断从依赖单梯绝对阈值的规则，改造成高精度优先的候选故障筛查算法。
- 将橡胶圈硬化专项诊断接入统一候选故障筛查链路，并使判定逻辑更贴合“曳引机本体安装振动传感器”场景下的支撑刚度/阻尼变化机理。
- 在更换电梯、安装位置或传感器幅值尺度变化时，降低分数漂移和误报风险，允许一定漏报但尽量不把正常工况报成异常。
- 为 Dify 展示链路补齐波形图生成能力，通过问答的形式可以获取到电梯的健康情况。


## 用户场景
- 维保或平台系统每天会定时采集几批 CSV，并需要生成“当前状态 / 当前风险 / 最近候选故障”的在线回复结果。
- 在同一套算法需要迁移到不同电梯或不同采集批次时触发本需求。
- 算法输出的是“疑似钢丝绳松动候选”，用于辅助维保筛查，不要求精确到松动几根钢丝绳。
- 算法输出的是“疑似橡胶圈硬化候选”，用于辅助维保筛查，不要求直接推断具体哪一个橡胶件失效。
- 在线问答不直接访问旧实时监控故障类型，而是查询最近一次定时批诊断结果。

## 传感器硬件前提
- 当前需求默认现场振动传感器为 `WT-VB01-485`，后续算法能力边界、接口说明和 Dify 文案应以该型号公开手册为准。
- 供电范围按参数表和接线定义采用 `5V~36V`；默认工作电压按 `12V` 估计；`12V` 工况工作电流约 `12mA`。若文档摘要与参数表冲突，优先以参数表和接线页为准，不使用摘要中的 `9V~36V` 说法作为正式约束。
- 通信接口为 `RS485`，波特率范围 `4800~230400bps`，默认 `9600bps`；高速度模式默认 `230400bps`。
- 设备对外输出为板载时间、三轴加速度、三轴角速度、三轴振动速度、三轴振动角度、三轴振动位移、三轴振动频率和芯片温度等处理后量，不应在需求或实现中将其表述为“高采样原始波形采集器”。
- 量程/范围按手册至少包括：振动速度 `0~50mm/s`、振动角度 `0~180°`、振动位移 `0~3000um`、振动频率 `1~100Hz`。
- 检测周期与截止频率均以 `1~100Hz` 为能力边界，默认 `10Hz`。当前需求中的趋势分析、候选筛查、在线状态和报告生成统一以 `fs≈40Hz` 的现场数据为基准；本次算法设计、调参、测试和验收均以 `40Hz` 数据成立为前提，不要求兼容其他采样频率场景。
- 环境与可靠性边界包括：工作温度 `-40°C~85°C`、存储温度 `-40°C~85°C`、抗冲击 `20000g`、防护等级 `IP67`。
- 安装方式至少支持磁吸安装和螺纹安装；默认螺纹规格按 `M8*1.25mm` 处理。
- 工程上应将 `WT-VB01-485` 视为“低频、边缘侧已处理量输出”的振动监测设备，适合旋转机械趋势监测、低频异常筛查和 RS485 现场接入；不适合在本需求中承担高频轴承诊断、冲击瞬态捕获、`100Hz` 以上细粒度频谱分析等角色。

## 范围
- 包含：围绕 `WT-VB01-485` 的低频输出能力，完善钢丝绳松动、橡胶圈硬化、定时批诊断、最新状态问答和波形图展示这几条现有链路的需求约束。
- 包含：补齐在线状态查询、离线 CSV 诊断、Dify 问答展示之间的边界，使“未上传 CSV 看最新状态，上传 CSV 看离线诊断”成为默认交互模式。
- 包含：明确 `fs≈40Hz` 采样前提、轴向映射、状态时效、CSV 字段要求等实现前提，减少后续实现歧义。
- 不包含：新增独立子系统、重写底层串口采集、重做训练流程、引入新依赖。

## 实现位置
- 在现有 `report` 诊断链路、批诊断链路、在线状态接口、Dify workflow 和对应测试范围内做渐进修改，不要求在本需求单中展开模块清单。
- 保持现有电梯监控、训练和报告主流程兼容，除满足本需求所需的最小接入外，不新增额外实现域。

## 接口与输入输出
- 输入继续以现有 CSV 诊断入口为主；默认要求输入能提供时间序列、三轴加速度、三轴角速度以及项目当前已支持的帧过滤信息。算法主路径只面向 `fs≈40Hz` 的 CSV 数据；缺失非关键列时允许降级处理，缺失关键列或采样率明显偏离 `40Hz` 时必须返回可解释错误或低质量结论，不能直接报 500。
- 波形图展示应继续兼容现有 `csv_path`、`csv_text` 或 `rows` 类输入方式，但“波形图”仅表示后端把已有输入可视化，不代表现场设备必须提供 `100Hz` 以上高频原始波形。
- 在线状态查询默认按 `elevator_id` 读取最近一次批诊断结果；未上传 CSV 时只走状态查询链路，上传 CSV 时走离线诊断和波形图链路。
- 最新状态输出至少应包含 `generated_at_ms`、`status`、`primary_issue`、`top_candidate`、`risk`、`report_markdown_draft` 和状态文件路径或等价定位信息。
- 在线状态接口必须返回状态是否过期的可解释信息；默认将超过 `24` 小时未更新的状态视为 `stale`，不得继续冒充“当前状态”。
- 最终用户可见结论继续保持保守，只输出 `normal`、`watch_only` 或 `candidate_faults` 语义，并保持 `primary_issue`、`top_candidate`、`candidate_faults`、`watch_faults` 等兼容字段。

## 业务规则
- 钢丝绳松动诊断应优先使用健康基线对少量核心特征做 robust 归一化，核心特征收敛为 `lateral_ratio`、`lat_dom_freq_hz`、`lat_low_band_ratio` 以及耦合聚合量（`ag_corr/gx_ax_corr/gy_ay_corr`），并以“特征证据值 + 命中数”完成判断。
- 钢丝绳通用异常辅助特征只保留 `a_rms_ac`、`a_p2p`、`g_std`；冲击类特征作为惩罚项。
- 当提供健康基线时，算法应尽量对整体幅值的等比例缩放保持近似不变。
- 当未提供健康基线时，算法应回退到仅依赖无量纲比值和相关性特征的保守模式，不得因缺少基线而报错。
- 触发逻辑应继续保守，优先保证正常工况不触发；允许漏报，但不能为了提高召回率显著增加误报。
- 单窗口除“摆动增强”外，还允许“横向失衡 + 耦合增强 + 低频成立 + 非冲击”的结构型证据通道，但不得再依赖大量分支阈值堆叠。
- 基线构建应尽量排除样本点过少的低质量窗口，避免健康画像被异常窗或空窗污染。
- 时间序列确认在遇到极低质量窗口时可跳过该窗，但不得因此把单点噪声放大成连续故障。
- 统一故障筛查总控不得直接把原始最高分等同于最终故障结论；应优先输出高置信候选故障，并保留原始排序供复盘。
- 当没有任何高置信候选时，总控应保持 `normal` 或 `watch_only` 状态，而不是强行给出唯一故障结论。
- 总体流程应明确分层为：`健康基线 -> 通用异常门 -> 类型判断（钢丝绳 / 橡胶圈）-> 最终决策层`，不得把单个专项脚本直接当作最终用户结论来源。
- 通用异常门只判断相对健康基线是否存在明显偏移，不直接输出故障类型；故障类型应由后续专项规则判断。
- 与横向/竖向相关的特征判断必须经过安装姿态映射。实现中不得默认传感器物理 `Z` 轴永远等于电梯竖向；若安装方向变化、重新贴装或更换电梯，应重建健康基线或显式传入轴向映射配置。
- 涉及窗口长度、频率统计和连续确认的实现统一按 `fs≈40Hz` 数据设计，允许使用与 `40Hz` 对应的固定时间窗和样本数假设；对于明显偏离 `40Hz` 的输入，不要求维持同等诊断能力。
- 钢丝绳专项规则应主要依赖横向比例、横向低频特征和加速度/角速度耦合聚合量等钢丝绳核心特征；共同异常特征只允许作为弱辅助。
- 钢丝绳专项应保持规则算法实现，不保留 centroid 模型训练、模型加载或规则-模型分数融合链路；跨梯验证只用于评估规则泛化性。
- 当前阶段橡胶圈规则保留实现但暂不参与统一主判；统一主判只保留钢丝绳主链。
- 钢丝绳专项主判应进一步收敛到频域与横向特征，至少保留 `lateral_ratio`、`lat_dom_freq_hz`、`lat_low_band_ratio` 作为主证据；耦合量仅作辅助解释。
- 批诊断应增加连续窗口确认：当连续多个窗口都命中钢丝绳观察级规则时，应允许把最新状态提升为已确认的钢丝绳连续异常。
- 橡胶圈专项规则应主要依赖 `energy_z_over_xy`、`az_p2p`、`az_cv`、`az_jerk_rms` 和轴间耦合变化等少量核心特征，并以“特征证据值 + 命中数”完成判断；共同异常特征只允许作为弱辅助。
- 共享异常特征只能支持 `watch_only` 或决策解释，不能单独把样本提升为某一类高置信候选。
- 最终决策层应输出统一主结论，并保留 `rope_primary`、`rubber_primary`、`system_abnormality` 和 `primary_issue` 等结构化字段供报告和接口消费。
- 当通用异常门判断为异常，但钢丝绳与橡胶圈专项规则都没有形成足够的类型专属性时，最终决策层应输出 `unknown` 作为观察级兜底主结论，而不是强行二选一。
- 波形图 API 应优先复用已有 CSV 解析和 `is_new_frame` 过滤逻辑，避免把补点直接绘成主波形。
- 诊断报告接口在请求显式开启时，应可自动附带波形图结果，供 Dify 报告展示使用。
- 橡胶圈硬化诊断应贴合“曳引机本体安装传感器”的观测条件，优先比较健康基线上的竖向响应增强、阻尼退化代理量增强、轴间耦合结构变化，而不是只依赖总振动绝对幅值。
- 橡胶圈硬化诊断中，竖向统计量（如 `az_std`、`az_rms_ac`、`az_p2p`、`az_cv`）、方向性代理量（如 `energy_z_over_xy`）、轴间相关结构（如 `corr_xy`、`corr_xz`、`corr_yz`）和瞬态/阻尼代理量（如 `az_jerk_rms`、`mag_std`、`mag_cv`）应参与综合评分。
- 橡胶圈硬化诊断应对冲击型样本保持抑制，避免把碰撞、急停或明显尖峰冲击误报成硬化。
- 当缺少健康基线时，橡胶圈硬化诊断可退回到更保守的方向性/竖向响应规则，但不应因为没有硬化模板样本就完全不可用。
- 定时批诊断应优先复用离线候选故障筛查算法，不应继续把旧实时 `fault_types.py` 里的通用故障规则当作在线主判来源。
- 定时批诊断应支持从目录中选取最近若干个 CSV 进行批处理，并把最近一次结果写入稳定路径，供在线状态查询接口读取。
- 最新状态中的风险分数应综合最近一次候选分数、最近多次重复出现的比例、分数趋势和数据质量，而不是沿用旧实时风险分数。
- 当最近一次结果为 `normal` 但此前多次存在候选时，风险应允许回落，但不能立即清零。
- 集成后的 Dify workflow 在未上传 CSV 时应仅查询最新状态接口，在上传 CSV 时应走离线诊断报告链路。
- 集成后的 Dify workflow 在未上传 CSV 且用户问题中包含电梯编号时，应优先按该 `elevator_id` 查询对应电梯的最新状态。
- 涉及 `WT-VB01-485` 设备能力的实现说明、接口文案和需求解释，均不得默认假设 `100Hz` 以上的原始高频采样、板载波形缓存或细粒度频谱分析能力。
- 本需求中的“波形图”应理解为基于已有 CSV 或传感器输出结果生成的可视化展示，不得反向推导为现场设备必须支持更高频的原始振动波形采集能力。
- 频域相关主证据（如 `lat_dom_freq_hz`、`lat_low_band_ratio`）默认建立在 `40Hz` 数据可稳定估计的前提上；当输入不满足该采样条件时，不得继续沿用相同阈值给出强结论。

## 验收标准
- 在当前 `fs≈40Hz` 现场输入下，离线诊断、定时批诊断、在线状态查询和 Dify 问答链路都能输出结构完整且字段兼容的结果。
- 需求落地后，接口文案、报告文案和规则说明不得再将 `WT-VB01-485` 表述为高频原始波形采集设备，也不得默认支持 `100Hz` 以上细粒度频谱分析。
- 当状态文件超过 `24` 小时未更新、文件缺失或批诊断失败时，在线状态接口必须返回可解释的 `stale` 或错误说明，而不是回退到旧实时故障结论。
- 当安装姿态变化或轴向映射变化时，系统必须要求重建基线或显式传入映射配置，不能直接沿用旧基线得出强结论。
- 当输入 CSV 的有效采样率明显偏离 `40Hz` 时，系统必须输出可解释的“非目标采样条件”提示或低质量结论，不得沿用 `40Hz` 阈值强行输出高置信候选。

## 测试用例
- 新增测试：在 `fs≈40Hz` 输入下，钢丝绳候选筛查、橡胶圈候选筛查、连续窗口确认和最新状态生成链路输出结构完整结果。
- 新增测试：当 CSV 缺失关键列时返回可解释错误；缺失非关键列时系统能够降级处理而非 500。
- 新增测试：最新状态文件超过 `24` 小时未更新时，在线状态接口返回 `stale` 标记或等价过期说明。
- 新增测试：上传 CSV 与未上传 CSV 两种 Dify 入口分别命中离线诊断链路和状态查询链路，不出现混用旧实时规则的情况。
- 新增测试：当输入 CSV 的有效采样率明显偏离 `40Hz` 时，算法返回低质量或非目标采样条件提示，而不是直接复用 `40Hz` 阈值输出候选故障。


## 非目标
- 不做基于监督学习的新模型训练。
- 不恢复旧版依赖硬化模板样本的实验脚本。
- 不将本次输出定义为“精确诊断松动几根钢丝绳”。
- 不做大量分支阈值堆叠式规则匹配；禁止使用大量超参数。
